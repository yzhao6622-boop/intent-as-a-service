# 端口已更新为 3002

## 已更新的文件

1. ✅ `src/server.ts` - 默认端口改为 3002
2. ✅ `env.example` - 示例端口改为 3002
3. ✅ `client/vite.config.ts` - 前端代理目标改为 3002
4. ✅ `启动说明.md` - 文档中的端口信息已更新
5. ✅ `README.md` - 文档中的端口信息已更新

## 需要您手动更新

请编辑 `.env` 文件，将端口改为 3002：

```env
PORT=3002
```

如果 `.env` 文件中没有 `PORT` 配置，系统会自动使用默认值 3002。

## 重启应用

更新 `.env` 文件后，重启应用：

```bash
# 停止当前运行的服务（Ctrl+C）
# 然后重新启动
npm run dev
```

## 新的访问地址

- 后端API：http://localhost:3002
- 前端界面：http://localhost:5174（或5173）
- 健康检查：http://localhost:3002/health

## 优势

使用新端口 3002 可以：
- 避免与占用 3001 端口的其他服务冲突
- 更稳定的开发体验
- 减少 nodemon 重启时的端口占用问题
